💡 핵심: OpenSearch는 "curl(명령어로 질문 보내기)"로 데이터를 확인합니다. 질문을 보내면 OpenSearch가 JSON(이름표 붙은 데이터)으로 답합니다. 처음엔 명령어로, 익숙해지면 웹 화면(Dashboards)으로 보면 됩니다. (맨 아래 참고)

📦 먼저, 데이터가 있는지 확인

1🔌
연결 확인OpenSearch가 살아있나?
$curl http://192.168.0.36:9200
▶ 이렇게 하면

OpenSearch가 "나 살아있어요"라고 자기소개를 답해줍니다.

1버전 번호와 이름이 적힌 JSON이 나오면 정상입니다.
2아무 응답이 없거나 "Connection refused"면 OpenSearch가 안 켜진 것 (docker ps 확인).
💡 192.168.0.36은 4070 머신 주소, 9200은 OpenSearch 포트입니다.
2🗂️
인덱스 목록데이터 칸이 어떤 게 있나?
$curl http://192.168.0.36:9200/_cat/indices?v
▶ 이렇게 하면

OpenSearch 안에 있는 데이터 서랍(인덱스) 목록이 표로 나옵니다.

1목록에 safety-zone-events가 보이면 우리 침범 데이터 서랍이 잘 만들어진 것.
2docs.count 칸 숫자가 지금까지 쌓인 침범 건수입니다.
💡 인덱스 = 엑셀 파일 한 개 같은 것. 우리 데이터는 safety-zone-events라는 인덱스에 들어갑니다.
3🔢
데이터 개수침범이 총 몇 건?
$curl "http://192.168.0.36:9200/safety-zone-events/_count?pretty"
▶ 이렇게 하면

지금까지 쌓인 침범 이벤트가 총 몇 건인지 숫자로 답해줍니다.

{ "count" : 47, ← 지금까지 47건 쌓임 "_shards" : { ... } }
💡 ?pretty를 붙이면 JSON이 보기 좋게 줄바꿈되어 나옵니다. 빼면 한 줄로 나와요.
4👀
최근 5건 보기실제로 뭐가 들었나?
$curl "http://192.168.0.36:9200/safety-zone-events/_search?size=5&sort=timestamp:desc&pretty"
▶ 이렇게 하면

가장 최근 침범 5건의 실제 내용을 들여다봅니다.

1_source 안에 {시각, 작업자번호, 거리, 심각도}가 들어있습니다.
2size=5는 5건만, sort=timestamp:desc는 최신순 정렬이란 뜻.
"_source" : { "timestamp" : "2026-06-30T11:22:01Z", "worker_id" : "W-3", "distance_cm" : 31.0, "severity" : "mid" }

🔎 이제, 검색·집계 (OpenSearch의 진짜 힘)

5📊
작업자별 집계누가 몇 번 침범했나?
$bash opensearch_check.sh 5
▶ 이렇게 하면

한 번의 질문으로 작업자별 침범 횟수와 가장 가까웠던 거리를 계산해줍니다.

1W-1은 몇 번, W-3은 몇 번… 자동으로 묶어서 세어줍니다.
2이게 집계(aggregation) — 수천 건도 순식간에 묶어 계산합니다.
✨ 이게 핵심! 일반 DB라면 직접 계산해야 하지만, OpenSearch는 "묶어서 세어줘"라고 질문만 하면 됩니다.
6🚨
고위험만 검색30cm 이내 침범만
$bash opensearch_check.sh 6
▶ 이렇게 하면

거리가 30cm보다 가까운 위험한 침범만 골라서 보여줍니다.

1range 검색 — "거리 30cm 미만" 같은 숫자 조건으로 거릅니다.
2가장 가까운 것부터 정렬되어 나옵니다.
7🔤
키워드 검색message에서 단어 찾기
$bash opensearch_check.sh 7
▶ 이렇게 하면

메시지 글 안에서 "침범"이라는 단어가 든 기록을 찾아줍니다.

1이게 전문 검색(full-text) — 구글처럼 글 속 단어를 찾는 기능.
2일반 DB는 이게 느리지만, OpenSearch는 검색엔진이라 빠릅니다.
💡 "error", "disconnect" 같은 다른 단어로 바꿔서도 검색해보세요.
8⏰
시간범위 검색최근 1시간 침범
$bash opensearch_check.sh 8
▶ 이렇게 하면

최근 1시간 안에 일어난 침범만 골라줍니다.

1now-1h = "지금부터 1시간 전까지"라는 시간 표현.
2now-20m으로 바꾸면 "20분 전 침범" 검색이 됩니다 (대시보드와 같은 원리).
✨ 대시보드의 "20분 전 침범자 찾기"가 바로 이 시간범위 검색을 쓴 것입니다.

🖥️ 명령어가 어려우면 — 웹 화면(Dashboards)으로 보기

curl 명령어 대신, 브라우저에서 마우스로 클릭하며 데이터를 볼 수도 있습니다.

  1. 브라우저에서 http://192.168.0.36:5601 접속
  2. 왼쪽 메뉴에서 Discover 클릭
  3. 인덱스 패턴으로 safety-zone-events 선택 (처음 한 번만 설정)
  4. 시간 막대그래프 + 침범 기록 목록이 화면에 보입니다
  5. 위쪽 검색창에 severity: high 같은 조건을 넣어 필터링
✨ 토론 팁: 여러 브라우저(라즈베리파이/NUC/PC)에서 Dashboards를 동시에 띄우고, 각자 다른 필터를 걸어 "나는 이렇게 보인다"를 비교하면 토론이 활발해집니다.

✅ 추천 실행 순서

처음이면 이 순서로 하나씩 해보세요:

1번(연결) → 3번(개수) → 4번(내용보기) 으로 "데이터가 있다"를 확인한 뒤, 5번(집계) → 8번(시간검색) 으로 "검색이 된다"를 체험하면 OpenSearch의 힘이 한눈에 들어옵니다.

⚠️ 데이터가 0건이면? 침범이 한 번도 없었거나 감지기가 안 돌고 있는 것입니다. NUC에서 python3 intrusion_detector.py --simulate로 가상 데이터를 좀 쌓은 뒤 다시 확인해보세요.
